📖 项目介绍

这是一个功能丰富的 Spring Boot 工具集管理系统，集成了多种实用的后端技术和主流云服务，旨在为开发者提供快速部署和二次开发的基础模板。

项目采用前后端分离架构，后端基于 Spring Boot 3.0+ 开发，前端使用原生 HTML/CSS/JavaScript，提供简洁直观的可视化界面，方便管理和操作各种服务。

🔧 核心功能模块

🔐

JWT 认证系统

支持管理员和用户双端认证，采用 JWT Token 实现无状态认证，提供完整的登录、Token 刷新机制。

💾

Redis 缓存管理

集成 Redis 实现高性能数据缓存，支持键值对的增删改查操作，适用于 session 管理、热点数据缓存等场景。

🔌

WebSocket 实时通信

基于 WebSocket 实现服务端与客户端的实时双向通信，支持消息推送、在线状态同步等功能。

☁️

阿里云 OSS 文件存储

集成阿里云对象存储服务，支持文件上传、下载、管理等功能，提供安全的云端存储解决方案。

💳

微信支付集成

集成微信支付 V3 API，支持创建支付订单、生成支付二维码、退款处理等完整的支付流程。

📚

API 文档中心

集成 Knife4j 在线 API 文档，自动生成接口说明，支持在线调试，让开发更加高效便捷。

🛠️ 技术栈

Spring Boot 3.0 Spring Security MyBatis MySQL57 Redis WebSocket JWT 阿里云 OSS 微信支付 V3 Knife4j

部署/移植教程（零基础可执行版）

目标：让一个没有后端经验的新手，也可以照着本文档把本项目功能部署起来，或者把功能迁移到自己的 Spring Boot 项目里。

建议严格按顺序执行：先跑通本项目，再做按模块移植。不要跳步骤。

1. 先跑通本项目（不要一上来就移植）

  1. 安装环境：JDK 17、Maven 3.8+、MySQL 8+、Redis 6+。
  2. 创建数据库：`tools_db`，并导入你项目对应的表结构和初始化数据。
  3. 修改 `application.properties` 中数据库、Redis、OSS、微信配置为你自己的值。
  4. 在项目根目录执行：`mvn clean package -DskipTests`。
  5. 启动后访问：`http://localhost:8080/static/index.html` 与 `http://localhost:8080/doc.html`。
# 启动命令（推荐）
mvn clean package -DskipTests
mvn spring-boot:run

5. 新手按步骤移植（照做即可）

  1. 先复制公共基础模块（Result、异常处理、ObjectMapper）。
  2. 复制配置类与 properties 类（`config` + `properties`）。
  3. 复制工具类（`utils`）和常量类（`constant`）。
  4. 复制各功能 controller 和需要的 mapper/entity/dto。
  5. 把 `application.properties` 与 `pom.xml` 补全。
  6. 编译：`mvn clean compile`，先解决编译错误再启动。
  7. 启动后逐模块验证（登录 -> Redis -> WebSocket -> OSS -> 支付 -> 文档）。
关键提醒：如果目标项目包名不是 `com.itcast`，请全局替换包名并修复 `@ComponentScan`、`@MapperScan`、`@ConfigurationProperties` 的扫描路径。

6. JWT 拦截器白名单（必须核对）

以下路径必须在拦截器放行，否则登录、文档、WebSocket 会出现 401 或无法访问。

/admin/employee/login
/user/user/login
/ws/**
/doc.html
/v3/api-docs/**
/swagger-resources/**
/webjars/**

7. 前端接入规范（复制到你自己的前端项目）

  1. 统一 API 地址变量（如 `API_BASE`）。
  2. 封装 `getAuthHeaders()`，统一注入 `Authorization`。
  3. 文件上传必须用 `FormData`，不要手动设置 `Content-Type`。
  4. WebSocket 地址按后端统一为 `ws://host:port/ws/message`。
  5. 所有响应按 `Result` 结构处理：`code === 1` 判成功。
  6. 商品缓存对比建议提供两个按钮组：`/admin/redis/product/native` 与 `/admin/redis/product/springcache`。

8. 商品缓存对比模块（Redis 原生 vs SpringCache）

该模块用于教学和移植演示：同一份商品数据，分别通过 Redis 原生 API 和 SpringCache 两种方式缓存。

后端新增接口：
GET    /admin/redis/product/native
DELETE /admin/redis/product/native/clear
GET    /admin/redis/product/springcache
DELETE /admin/redis/product/springcache/clear

关键实现点：
1) Redis 原生：RedisTemplate 手动 set/get/del（自定义 key）
2) SpringCache：@Cacheable + @CacheEvict + @EnableCaching
3) 数据源：product 表（建议先准备测试数据）
验收标准：首次加载 source 显示 `database`，再次加载显示缓存来源；清理缓存后重新加载应再次回到 `database`。

9. 一键验收清单（迁移完成后）

  1. 管理员登录成功，token 能正常鉴权。
  2. Redis 的 set/get/delete 全部可用。
  3. 商品缓存“Redis 缓存 / SpringCache缓存”都可加载并正确显示来源。
  4. WebSocket 可连接、收发消息正常。
  5. OSS 上传成功并返回可访问 URL。
  6. 支付下单接口可返回二维码或支付信息。
  7. Knife4j 页面可打开且接口列表正常展示。

10. 常见报错与解决（新手高频）

  • 文档请求异常：检查 `/v3/api-docs` 是否能返回 JSON；`WebMvcConfiguration` 中不要让自定义转换器覆盖 springdoc 响应。
  • 启动报 Bean 找不到：通常是文件没复制全，优先检查 `properties`、`config`、`utils`、`mapper`。
  • 401 未授权：检查拦截器白名单、token 是否放在 `Authorization` 请求头。
  • Redis 报连接失败：检查 Redis 是否启动、端口是否 6379、密码和 database 是否一致。
  • SpringCache 不生效：确认主启动类已加 `@EnableCaching`，并且是通过 Spring 容器调用 `@Cacheable` 方法。
  • 上传失败 415：前端上传时不要手动设置 `Content-Type`。
  • 微信支付失败：重点核对商户号、证书路径、`apiV3Key`、回调 URL 是否公网 HTTPS。
